It is better if we all know what a graph is!
  1. Dots are vertices, connections are edges
  2. The degree of a vertex is the number of neighbors it has
  3. The distance between two vertices is the length (number of edges) of the shortest path connecting them;
  4. The diameter is the maximum distance, that is, $\max_{u,v}{\rm dist}(u,v)$;

  • Graph theory traces its roots to the 18th century.
  • Founded by the Swiss mathematician, Leonhard Euler.
  • Started with the solution to the Königsberg Bridge Problem in 1736.

: The Königsberg Bridge Problem

  • Königsberg was built around the Pregel River.
  • It included two large islands connected to each other and the mainland by seven bridges.
  • Challenge: Find a walk through the city crossing each bridge once and only once.

: Euler's Representation

Euler's approach to solving the problem:
  • Represent the land masses as nodes (or vertices).
  • Represent the bridges as edges (or arcs).
  • Resulted in the creation of a graph.

: Euler's Insight

Using the graph:
  • Euler noted the number of bridges touching each land mass.
  • Proved that for a walk to cross each bridge once:
    • All vertices (except possibly the start and end) must have an even number of edges.
  • Königsberg did not meet this criteria.

  • Euler's solution to the Königsberg Bridge Problem is recognized as the formal beginning of graph theory.
  • It showcased the potential of representing and solving real-world problems using graphs.

Using graphs to model the world!

  1. Vertices can be people and edges represent friendship (social networks)
  2. Vertices can be power stations and edges are wires connecting them
  3. Vertices can be neurons and edges are connections between them
  4. In general, a graph can be used to model set of things with relation between those things.

In real-life, networks can be governed by some randomness and can change over time.

So we need graphs that are random (that possibly change over time).

Random Graphs

  1. Static models. We add a probability distribution over a set $\mathcal{G}$ containing graphs and sample from it.
    Examples: Erdős-Reyni, Configuration model, Strogattz-Watts;
  2. Dynamic models. We can inductively change our graphs according to some random rule. This generates a vector or a sequence of random graphs. Barabasi-Albert, Dynamic Configuration Model.

Erdős-Reyni $G(n,p)$
  1. Start with $n$ vertices with no degree;
  2. Connect each pair of vertices with probability $p$ and independently of each other.
 Uniform Attachment
  1. At each time step $n$, generate $G_n$ from $G_{n-1}$ as follows:
    1. Add a new vertex $v_n$ to $G_{n-1}$.
    2. Connect $v_n$ to a uniformly selected vertex $u \in G_{n-1}$.
  2. We obtain a Markov Chain $\{G_n\}_{n \in \mathbb{N}}$.

A model to Networks: The Preferential Attachment model

  1. At each time step $n$, generate $G_n$ from $G_{n-1}$ as follows:
    1. Add a new vertex $v_n$ to $G_{n-1}$.
    2. Connect $v_n$ to a vertex $u \in G_{n-1}$ with probability proportional to its degree, that is $$ P(v_n \leftrightarrow u | G_{n-1}) = \frac{\text{degree}(u)}{\sum_{v \in G_{n-1}}\text{degree}(v)} $$
  2. We obtain a then obtain a sequence $\{G_n\}_{n \mathbb{N}}$.

This is the so-called Barabási-Albert Model. Published in the journal Nature and has more than $45k$ citations.

The BA-model is an interesting example of a good model that lies in the intersection of Computer Science and Mathematics.
  1. It is a simple and easy to describe model.
  2. It is capable of capturing a lot of graph properties observed from empirical data.
  3. It has the mathematical challenge mathematicians care about: its dynamics create a strong dependence between the vertices, which wasn't observed in other random graph models like the $G(n,p)$. This makes its analysis harder.
  4. Initially, (2) had been shown only via simulations or nonrigorous results. Over the years, mathematicians proved interesting properties about it and its many variants.

Simulation v.s.Theory

Holme-Kim Model

Fix $p\in [0,1]$ and a positive integer~$m>1$. The evolution from $G_{t-1}$ to $G_t$ consists of the addition of a new vertex $v_t$ and $m$ new edges between $v_t$ and vertices already in $G_{t-1}$. These $m$ edges are added sequentially in the following fashion.
  1. The first edge is always attached following the preferential attachment (PA) mechanism, that is, it connects to a previously existing node $w$ with probability proportional to the degree of $w$ in $G_{t-1}$.
  2. Each of the $m-1$ remaining edges makes a random decision on how to attach.
    1. With probability $p$, the edge is attached according to the triad formation (TF) mechanism. Let $w'$ be the node of $G_{t-1}$ to which the previous edge was attached. Then the current edge connects to a neighbor $w$ of $w'$ chosen with probability proportional to number of edges between $w$ and $w'$.
    2. With probability $1-p$ ($p\in [0,1]$ fixed) the edge follows the same PA mechanism as the first edge (with fresh random choices).

: Global Clustering Coefficient

Given a graph $G$, we define its global clustering coefficient as c(G) = \frac{3 \times \text{ # of triangles in }G}{\# \text{ of potential triangles}}

The global clustering coefficient captures the tendency that some network have of closing triangles.

Friends of friends tend to be friends.

: Global Clustering Coefficient

In the paper Generalized preferential attachment: Tunable power- law degree distribution and clustering coefficient (a CS paper) the authors investigated via simulations many variants of the BA-model, including the HK-model. From their simulation, they concluded that regarding the $HK$-model, $c(G_t)$ satisfies $$ \liminf_{t \to \infty }c(G_t) > 0, $$ almost surely.

: Global Clustering Coefficient

However..... R. I. Oliveira, R.R., R. Sanchis - Advances in Applied Prob. For any $p \in [0,1]$, $m>1$ $$ \lim_{t \to \infty} c(G_t) = 0, $$ almost surely. More specifically, there exist constants $b_1$ and $b_2$ depending on $p$ and $m$ only, such that $$ P\left( \frac{b_1}{\log \log t} \le c(G_t) \le \frac{b_2}{\log \log t} \right) = 1 - o(1). $$

Generating Graphs with Random Walks:Tree Builder Random Walk

  • The Barabási-Albert model and its variants proved to be capable of capturing properties we do observe in real-life networks
  • However, computer scientists have tried to address its global nature. In the model, the new vertex has a global knowledge of the network, which is not realistic.
  • In the recent years, the community has made efforts to come up with new models that are local in nature, but have the same nice properties captured by Preferential Attachment models.

J. Englander, G. Iacobelli, G. Pete, R.R. - Annals of Applied Prob. There exist the right choice for the "coin-parameter" so that the graphs generated by the Barabási-Albert model and the graphs generated by the Tree Builder are essentially the same.
